VCD Analyzer

In FlexPRET, the success or failure of a 
delay_until_period();
result in two different patterns in the VCD trace. Those patterns are impossible to detect using VCD analyzers because one cannot write a query in pure combinatory logic (state-based).

Solution: write a small state-based program to detect the two patterns (success and fail). 

var vcd = require('vcd');
var fs  = require('fs');
fs.createReadStream('./multi.vcd')
  .pipe(vcd.createStream())
  .on('sample', function (index, changes, last) {
    // sequential program with state memorization
    switch(state) {
      case State.Failure_Pattern_Entering:
        // ...
        break;
      // ...
    }
  })
  .on('finish', function () {
    console.log("output result");
  });

Common libraries

Libraries for UI

  • Framework HTML+CSS+JS (Frontend)
    • Out-of-the-box
    • Highly customizable
    • Same output on all browsers (including IE)
    • Handle presentation, user only cares about content
    • Mostly intended for web sites
    • Materialize
    • Bootstrap
    • Foundation
  • Eclipse-like apps (Frontend)
    • Window and tab managements
    • Easy to configure
    • Require a bit of JS knowledge to work
    • Golden Layout
    • ...
  • Multi-platform apps (Frontend + Backend)
    • Multi-device
    • Unique API for queering device sensors
    • Trickier to configure
    • Apache Cordova
    • ...

Libraries for Data visualization (non-exhaustive list)

  • Charts and tables
    • Highly customizable
    • Large amount of data
    • Sometimes tedious to configure (specially ChartJS)
    • ChartJS
    • SlickGrid
  • Graphs
    • Multitude of possibilities
    • Highly interactive
    • Sometimes tedious to configure
    • draw.io
    • Fabric.JS
    • Graphviz
    • D3.js
    • PlantUML

Libraries for Code editing & visualization (non-exhaustive list)

  • Parsing
    • Online editor to test grammars
    • Can be used to parse and/or evaluate
    • Can be embedded in a JS editor
    • PEG.js
  • Highlighting
    • Large set of programming languages + custom
    • highlight.js
    • Prism
  • Editing
    • Large set of programming languages + custom
    • Highly configurable
    • Include highlighting support
    • Include smart features: code snippet, semantic checking, etc.
    • Painful to configure new languages
    • Ace
    • CodeMirror
